Git/Github 環境 を導入する

ローカルに Git をインストール

Linux なら sudo apt install git-all ではいる。

windows なら Git の公式サイトから、 インストーラーを落としてくる。管理者権限がないときは、ポータブル版の Git を使えばいい。

インストール先を聞かれる。どこでもいいが、たとえば ~/bin/Git/（ポータブル版なら~/bin/PortableGit）あたりを指定する。

git.exe は、~/bin/Git/cmd/（ポータブル版なら ~/bin/PortableGit/cmd/）にある。そこに PATH を通しておきます。

さらに自分の PC での設定：

$ git config --global user.name iotsuka1958 # GitHub のユーザ名
$ git config --global user.email smiley.at.circus@gmail.com # 同メアド
$ git config --global pull.rebase false
$ git config --global core.editor xyzzycli # 自分の好きなエディタを指定（ゆくゆくは emacsclient）
$ git config --global init.defaultBranch main # Git 2.28以降
$ git config --global grep.lineNumber true # git grep 実行時に行番号を表示

上記の設定は ~/.gitconfug に書き込まれる。ねんのために、ちゃんと設定されたか確認します。

$ cat ~/.gitconfig

プロキシサーバーが設定されていると、うまくつながらないので、ひとてま必要。ターミナルで Git にプロキシを設定します：

$ git config --global http.proxy (プロキシサーバーのアドレス：ポート番号)

プロキシサーバーの情報は、windows なら ターミナルで次のコマンドをうつとわかります。 

$ netsh winhttp show proxy

Github 上にリモートのリポジトリを作成・ひもづけ

GitHub でリポジトリをつくります（からっぽでもいいそうです）。それをクローンします： 

git clone https://github.com/username/repositoryname.git
